# Introduction

Welcome. This course teaches Rust by building small things, one at a time, until you've built enough to build anything.

## What you'll do in every lesson

1. Read a short chapter (this book).
2. Do a few **exercises** — small broken snippets you fix. Instant feedback via `rbb watch`.
3. Build a **project** for the lesson — a working program, small enough to finish in one sitting, with tests you run to confirm you got it right.

Bigger **capstone projects** bookend each phase. Those are the pieces you'll keep referring back to, and that connect directly to the TLS and Async courses if you take them later.

## The rules of the course

**You cannot search the internet.** You have the Rust book, the standard library docs, and `rustc --explain` — all served locally. This is a feature, not a limitation. Rust has one of the best error messages + one of the best docs of any language. Learning to navigate them is the skill.

**The compiler is your friend.** Rust's errors tell you what to do. Read them. Treat `rustc` as a tutor, not an obstacle.

**Type things out.** Copy-pasting code from the book into your editor and running it teaches you nothing. Retype it. You'll feel slow for a week. After that you won't.

## When you're stuck

In this order:
1. Read the error message from the bottom up.
2. `rustc --explain E0382` (or whatever code).
3. Search the offline Rust book — it's linked from your server's local docs page.
4. Ask your admin.

Good luck.
